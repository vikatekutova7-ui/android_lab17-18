- Имитация ошибок загрузки с корректной обработкой
- Отображение прогресса загрузки
- Вычисление индекса погоды на основе полученных данных
- Адаптивный интерфейс на Jetpack Compose

## Контрольные вопросы

### А) В чём разница между launch и async?

**launch** — запускает корутину и не возвращает результат. Используется для операций, где результат не нужен.

```kotlin
viewModelScope.launch {
    saveDataToDatabase()
}

```
**async** — запускает корутину и возвращает Deferred<T>, из которого можно получить результат через await().

```kotlin
val result = viewModelScope.async {
fetchDataFromNetwork()
}.await()
```
**Когда использовать:**

- launch — для операций, где не нужен результат (логирование, отправка уведомлений)

- async — когда нужно получить результат (загрузка данных, вычисления)

### Б) Что такое suspend функция?
**Suspend-функция** — это функция, которая может приостанавливать своё выполнение без блокировки потока.

**Может ли она вызываться из обычной функции?**
Нет, suspend-функции могут вызываться только из других suspend-функций или из корутины.

**Почему delay() не блокирует поток?**
delay() — это suspend-функция, которая приостанавливает только корутину, освобождая поток для выполнения других задач. В отличие от Thread.sleep(), который блокирует весь поток.

### В) Зачем нужны разные диспетчеры?
Диспетчеры определяют, на каком потоке будет выполняться корутина.

| Диспетчер | Когда использовать | Пример |
|-----------|-------------------|--------|
| Dispatchers.Main | Обновление UI | Изменение текста в TextView |
| Dispatchers.IO | Ввод/вывод | Загрузка из сети, работа с БД |
| Dispatchers.Default | Тяжёлые вычисления | Обработка больших списков, сортировка |

**Что будет, если выполнить тяжёлое вычисление на Dispatchers.Main?**
Приложение зависнет (ANR — Application Not Responding), так как UI-поток будет занят вычислениями и не сможет обрабатывать пользовательский ввод.

### Г) Что произойдёт, если не обработать исключение в корутине?
Исключение в корутине без обработки приведёт к крашу приложения (FATAL EXCEPTION).

**Как корректно обрабатывать ошибки?**

```kotlin
viewModelScope.launch {
    try {
        val data = repository.fetchData()
    } catch (e: Exception) { 
        _state.value = _state.value.copy(error = e.message) 
   }
}
```
**Зачем нужен try-catch внутри launch?**
Чтобы перехватить исключения и корректно обработать их, не допуская краша приложения.

### Д) Как работает автоматическая отмена корутин?
**Что такое viewModelScope?**
**viewModelScope** — это встроенная область видимости корутин, привязанная к жизненному циклу ViewModel.

**Когда корутины отменяются автоматически?**
Корутины, запущенные в viewModelScope, автоматически отменяются при вызове onCleared() ViewModel (когда Activity или Fragment уничтожается).

```kotlin
class WeatherViewModel : ViewModel() {
    init {
        viewModelScope.launch {
            loadWeatherData()
        }
    }
}
```
